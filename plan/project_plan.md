# 项目总体计划 - Poker Live Assistant

## 项目目标

在不增加服务器成本前提下，快速落地一个手机可用的德州扑克实时辅助工具，覆盖翻前到河牌的实时建议与解释。

## 技术栈建议（简化且稳妥）

- **客户端**：React + TypeScript + Vite（PWA）
- **状态管理**：Zustand
- **本地存储**：IndexedDB（Dexie）+ localStorage
- **算法**：TypeScript 实现 Monte Carlo
- **自然语言**：
  - 默认：模板 + 关键词解析
  - 可选：MiniMax API

## 模块拆分

### plan_02 牌桌与玩家模块
- 牌桌人数、盲注、座位、玩家映射、补码
- 玩家画像与历史行为统计

### plan_03 行动与牌局状态模块
- preflop/flop/turn/river 状态机
- 行动事件流与底池计算

### plan_04 自然语言解析模块
- 文本转结构化行动事件
- 模板解析优先，MiniMax 作为增强通道

### plan_05 概率与胜率计算模块
- 主引擎（快速近似）
- 备引擎（高精度）

### plan_06 策略建议与解释模块
- 输出 fold/call/raise/all-in 建议
- 解释因子：位置、底池赔率、玩家倾向、范围强弱

## 执行阶段

1. **阶段A（1周）**：完成基础数据模型、状态机、UI 骨架
2. **阶段B（1周）**：完成主引擎与基础建议
3. **阶段C（1周）**：接入备引擎与玩家记忆上下文
4. **阶段D（3-5天）**：自然语言增强、压测和体验优化

## 成本与部署策略

- 默认单机运行，不依赖后端
- 仅在需要跨设备同步时，再加免费后端（如 Supabase 免费层）
- MiniMax 仅用于解析增强，可手动开关，控制额度

## 风险与缓解

- **风险**：现场输入过慢  
  **缓解**：快捷动作按钮 + 语音/文本快捷解析
- **风险**：胜率计算耗时  
  **缓解**：双引擎并行，先快后准
- **风险**：行为数据噪声大  
  **缓解**：对玩家画像设置置信度与衰减机制
